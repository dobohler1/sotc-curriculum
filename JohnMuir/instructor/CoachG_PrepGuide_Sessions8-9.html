4. Session 8: Sensor Recon — Coach G's Playbook

The single most important thing this session: The sensor code introduces while True: and if statements inside a loop. This is the biggest conceptual jump in the program so far. Do not rush the explanation. Walk through the loop line by line on slides 6 and 7. Make sure students understand what "check, check, check, STOP" means before they start typing.

Before Students Arrive

  • Open Session8_SensorRecon.html in Chrome/Safari, go full screen (F11 or Cmd+Shift+F)
  • Make sure robots have distance sensors plugged in. Note which port for each robot (likely port F, but check). Write the port letter on a sticky note on each robot.
  • Have 2–3 books or boxes ready for demo. You'll use one to show the robot stopping.
  • Post Assignment 9 in Google Classroom if not already done

Session 8 Troubleshooting

ProblemLikely CauseFix
Robot drives but never stops Wrong port in code Check which port the distance sensor is plugged into. Change port.F to match (e.g., port.D).
Robot stops immediately without driving Sensor reading something right away (table surface, student's hand) Clear the path in front of the sensor. Make sure nothing is within 15cm at start. Raise the robot if it's reading the table.
"distance_sensor is not defined" Missing import Add import distance_sensor at the top. It's separate from motor_pair.
"port is not defined" Missing import Change the first line to: from hub import light_matrix, sound, port
Robot drives forever, code seems frozen Missing await runloop.sleep_ms(50) in the while loop Without the sleep, the loop runs too fast and blocks. Add the sleep line inside the while loop.
Indentation error on the if line Mixed indentation inside while loop Everything inside while True: needs 8 spaces (4 for main + 4 for while). Everything inside if needs 12 spaces.
Sensor detects obstacle but robot doesn't stop motor_pair.stop() missing or in wrong place Make sure motor_pair.stop(motor_pair.PAIR_1) is inside the if block, before the break.
Indentation cheat sheet for this session's code:
0 spaces: imports, motor_pair.pair(), runloop.run()
4 spaces: everything inside async def main():
8 spaces: everything inside while True:
12 spaces: everything inside if dist != -1 and dist < 150:

5. Session 9: The Gauntlet — Coach G's Playbook

The single most important thing this session: Set up the course before students arrive. If you're building the course during class, you lose 5–10 minutes of work time. The course doesn't need to be fancy — 2–3 books or boxes in a line on the floor, with gaps between them. The point is that the robot has to navigate around real obstacles.

Before Students Arrive

  • Open Session9_TheGauntlet.html in Chrome/Safari, go full screen
  • Set up the course: On the floor (not a table — robots will drive off the edge). Use books, boxes, or cones. 2–3 obstacles with ~18 inches of open space between them.
  • Mark the start zone with tape or a piece of paper
  • For Green Table: place a red piece of paper at the finish zone (for color sensor detection)
  • Post Assignment 9 in Google Classroom if not already done
  • Have your scoring sheet ready (below)

Course Setup Guide

Keep it simple. A straight-ish path, about 4–5 feet long, with 2–3 books standing upright as obstacles. The obstacles should be big enough for the distance sensor to detect (a textbook is perfect — a pencil is not). Leave enough room between obstacles for the robot to reverse and turn. If the course is too tight, no one clears it and frustration wins.

Recommended layout:

  1. Start zone: Tape line on the floor. Robot starts here facing forward.
  2. Obstacle 1: A textbook standing upright, about 18 inches from start. Centered in the path.
  3. Obstacle 2: Another book, about 18 inches past obstacle 1, offset slightly to one side.
  4. Obstacle 3 (optional): A third book, 18 inches past obstacle 2, offset to the other side.
  5. Finish zone: Red piece of paper on the floor at the end (for Green Table's color sensor). For everyone else, just reaching this area counts.

10–12 7 Show table tasks. Students move to seats. Red Table types starter code. Yellow Table types and modifies. Green Table builds from scratch.

22–32 11 Course testing time (10 min). Students bring robots to the course for test runs. Line them up. Quick test runs. Let students iterate — run, observe, go back and change a number, run again. This is real engineering.

Session 9 Troubleshooting

ProblemLikely CauseFix
Robot avoids first obstacle then drives into the second Code only runs the detection loop once Make sure the avoidance logic is inside a for loop (like the starter code) so it keeps checking after each avoidance.
Robot keeps turning in circles Always turns the same direction, obstacle is still in the way Reverse more before turning, or alternate turn direction. Or increase the turn angle.

Robot completes course but code errors at the end Loop runs out or tries to access sensor after disconnect Not critical — if the robot finished the course, that's a success. The error is cosmetic.

6. Managing Engagement Across Both Sessions

Likely Challenges

The "it's too hard" moment: The sensor code is the longest and most complex code students have typed. Some Red Table students will hit a wall at the while True: loop. When this happens, don't re-explain the concept — sit next to them, look at their screen, and find the one thing that's wrong. It's almost always indentation or a wrong port letter. Fix that one thing, run the code, and let the working robot carry the momentum.
Vincent and Winston in Sessions 8–9: These two will finish the basic sensor code in 5 minutes. Redirect them immediately: (1) Write their own avoidance algorithm, (2) Add the color sensor, (3) Help a Red Table student get their code running (but point and explain, don't type for them). In Session 9, they should demo first to show the class what a full course run looks like.

If a Student Was Absent for Session 8

They missed the sensor introduction. In Session 9:

  1. Give them the starter code directly (don't make them type from the slides)
  2. Spend 2 minutes explaining: "This code drives forward and checks the distance sensor in a loop. When something is close, it stops and turns."
  3. Have them run it, see it work, then start modifying
  4. They'll learn faster from running working code than from a lecture

7. Quick Reference: Sensor Commands

Keep these handy during the session for quick student help:

# Distance sensor
from hub import port
import distance_sensor
dist = distance_sensor.distance(port.F)  # returns mm, -1 if nothing

# Color sensor
from hub import port
import color_sensor
color = color_sensor.color(port.E)
# 0=Black, 1=Violet, 3=Blue, 4=Cyan, 5=Green, 7=Yellow, 9=Red, 10=White

# Continuous drive (doesn't stop on its own)
motor_pair.move(motor_pair.PAIR_1, 0, velocity=40)

# Stop motors
motor_pair.stop(motor_pair.PAIR_1)

# Drive for a set time
await motor_pair.move_for_time(motor_pair.PAIR_1, 2000, 0, velocity=50)

# Reverse
await motor_pair.move_for_time(motor_pair.PAIR_1, 1000, 0, velocity=-40)

# Turn right
await motor_pair.move_for_time(motor_pair.PAIR_1, 800, 100, velocity=30)

# Turn left
await motor_pair.move_for_time(motor_pair.PAIR_1, 800, -100, velocity=30)
